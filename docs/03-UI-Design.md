# 03 - UI Design (V1)

## 1. UI 设计理念

本系统的 UI 设计遵循 **“平静界面”** 原则，强调信息本身而非容器。设计哲学可概括为：

- **克制界面**：避免鲜艳色彩和过度装饰，降低视觉压强，让用户专注于内容阅读。
- **信息优先**：每一个像素的存在都应服务于信息传递，不为了美观而添加无效视觉元素。
- **极低视觉噪音**：无背景纹理、无渐变、无投影泛滥，边框和分割线使用克制。
- **可维护性优于美学**：设计风格统一、规则简单，确保未来新增城市或内容时，无需重新设计视觉规范。

---

## 2. 布局结构

整体采用经典的 **“侧边栏 + 主内容区”** 双栏布局，层级清晰：
+------------------+------------------------------------------+
| 侧边栏导航 | 主内容区域 |
| (Sidebar) | (Content Area) |
| | |
| 🏠 总览 | [分类标题] |
| 🍜 美食 | ────────────────── |
| 🛍 购物 | | 卡片 1 | 卡片 2 | 卡片 3 | |
| 🏨 酒店 | | 卡片 4 | 卡片 5 | 卡片 6 | |
| 🚇 交通 | |
| ☕ 咖啡馆 | (内容以网格或列表形式呈现) |
| 💆 按摩 | |
| 🏛 景点 | |
+------------------+------------------------------------------+


- **顶部栏**：极简设计，仅放置系统名称（如“曼谷指南”）和必要的城市切换下拉入口，不占用主要视觉权重。
- **侧边栏**：承载全部一级导航（城市 → 分类），是用户浏览的起点。
- **主内容区**：动态展示当前分类下的条目卡片，内容清晰，留白充足。

---

## 3. 导航设计

导航逻辑遵循 **“先选城市，再选分类”** 的二维路径，但交互上保持扁平：

- **侧边栏主导航**：所有分类（美食、购物、酒店等）在侧边栏中以列表形式平铺展示，用户点击即切换内容，无需多级展开。
- **无复杂路由**：每个页面对应一个明确的“城市+分类”组合，URL 结构简单（如 `#bangkok/food`），但无需前端路由库支持。
- **一致的结构**：无论切换到哪个分类，主内容区的布局方式完全一致（标题区 → 卡片网格），降低用户学习成本。
- **当前状态明确**：侧边栏中当前激活的分类使用细微的样式变化（如文字加粗或左侧小竖条）进行标识，确保用户不迷路。

---

## 4. 视觉风格体系

**色彩哲学**
- 主色调：暖灰（`#f5f3f0` 背景）搭配深灰（`#2c2c2c` 文字）。
- 强调色：低饱和度的芥末黄或陶土色，仅用于少量关键交互元素（如当前激活状态、链接），占比不超过 5%。
- 中性色：卡片背景纯白（`#ffffff`），边框使用极浅灰（`#e8e6e3`），整体呈现柔和、温暖的纸质阅读感。

**排版**
- 字体：使用系统默认字体（`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`），不引入外部字体，保证加载速度和跨平台一致性。
- 字号层级：仅设三级 —— 标题（20px）、正文（15px）、辅助信息（13px），比例克制。

**间距与卡片**
- 遵循 8px 基准网格，所有内边距和外边距为 8 的倍数（8、16、24、32px），确保视觉韵律一致。
- 卡片样式：纯白背景，无阴影，仅保留极浅的底部边框（1px）或圆角（4px），用于区分不同条目，避免视觉堆叠感。

---

## 5. 组件设计

**侧边栏菜单**
- 纯文本列表，每个项由图标（可选，使用 Unicode 符号）和分类名称组成。
- 悬停时轻微背景色变化（`#f0eeeb`），点击后无复杂动效，仅更新激活状态。

**分类页面标题区**
- 包含分类名称（如“美食”）、简短描述（如“记录曼谷值得再去的餐厅与街头味道”）以及条目总数（如“共 12 处”），让用户快速了解该分类概况。

**条目卡片**
- 每个卡片包含：名称、地址（简写）、个人评分（星级或数字）、访问日期、简短标签（如#本地、#夜市）。
- 卡片内容左对齐，信息密度适中，不展示大图，仅保留最核心的参考字段。
- 卡片在网格中均匀排列，宽度自适应，高度由内容撑开，保持自然。

**辅助组件**
- 搜索框：放置于主内容区顶部，仅对当前分类进行客户端文本过滤，不做全局复杂搜索。
- 排序切换：简单的下拉或按钮组（如按评分、按时间），控制在 2~3 个选项以内。

---

## 6. 响应式设计

- **桌面端（≥ 1024px）**：固定侧边栏宽度（240px），主内容区占据剩余宽度，卡片以 3 列网格呈现。
- **平板端（768px ~ 1023px）**：侧边栏宽度收缩至 200px，卡片调整为 2 列。
- **移动端（< 768px）**：
  - 侧边栏默认隐藏，通过左上角的汉堡菜单按钮唤出（覆盖层或滑入式）。
  - 主内容区占满屏幕宽度，卡片改为单列排列，确保手指触控友好。
  - 顶部栏精简，仅保留菜单按钮和当前分类名称。

---

## 7. 信息设计原则 —— 每页只回答一个问题

每个分类页面承担且仅承担一个明确的用户任务，内容组织围绕该任务展开：

| 页面 | 核心用户问题 |
|------|-------------|
| 美食 | 去哪儿吃？ |
| 酒店 | 住哪儿好？ |
| 交通 | 怎么走？ |
| 景点 | 看什么？ |
| 按摩 | 去哪儿放松？ |
| 购物 | 买什么/去哪儿买？ |
| 咖啡馆 | 去哪儿坐坐？ |

每个页面的字段设计（如美食页面突出“推荐菜”，交通页面突出“费用/时长”）均围绕该核心问题定制，不混合不同目的的信息。

---

## 8. UI 约束

为确保长期可维护性和轻量化，明确限定以下边界：

- ❌ **不使用任何 UI 框架**（React、Vue、Angular 等）—— 保持纯静态 HTML + CSS + 少量原生 JavaScript。
- ❌ **无复杂动画** —— 页面切换为直接显示/隐藏，无过渡动效；悬停效果仅使用 CSS `transition: background 0.15s` 级别。
- ❌ **无重视觉特效** —— 无粒子、无视差滚动、无渐变背景、无模糊效果。
- ❌ **无不必要 UI 复杂度** —— 不引入标签页、手风琴、弹窗等交互组件，除非绝对必要。

---

## 9. 设计目标总结

本 UI 设计的最终目标可以概括为：

> **简单但不简陋** —— 功能完整，视觉优雅且克制，每个元素都有其存在理由。
>
> **结构化但不死板** —— 布局规范统一，但卡片内容可根据分类灵活调整字段。
>
> **干净且可维护** —— 样式规则一致，新增城市或分类无需重构样式。
>
> **令人愉悦但不依赖装饰** —— 舒适感来源于排版、留白和秩序，而非多余的视觉点缀。

我们希望用户在打开这个页面时，感受到的是一份**可信赖的个人手稿**，而不是一个商业旅行平台。
